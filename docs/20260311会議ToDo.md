タカショーデプロイ環境MTG

- ドキュメント化することでコーディング・デプロイ時に最適化は可能
- もし手動でデプロイが必要な場合は簡単なものでもいいのでマニュアル化してほしい→タカショーデジテック（クライアント側）大下さんが作成してくれる。


アンバサダーの皆さんはアプリを作る時点で、JANDIにて大下さんに開発環境欲しいと伝えると空の環境を作ってくれる
そのGitHubにPushしたら自動で更新さえるイメージ（Vercelとかと同じ）
GitHubを利用するには2段階認証が必要なのでキーパスXCというもの入れて対応する人もいる

context が溢れる場合の対応を明示する。ClaudeCodeのcontextの円グラフの進捗をどう解釈してどうコンテキストの溢れを避けるか、100％に近づくとどうなってどう対応すべきかを明確に振れるセクションが欲しい。

人と初心者に向けてしっかりと分かる、非エンジニア、非技術者でも分かるような内容でちゃんと咀嚼してくださいね。ちゃんとアクションレベルでここを押せばこうなるんだとか、そういうのを明確にお願いします。

さらにAIDDキットという形で環境の最小要件ではあるけれども、CLAUDEコードをうまく使えていくためのディレクトリもプリセット的に用意しているので、それをうまく活用してもらえたらみたいな感じで訴求するセクションもあったら嬉しいです。

Manifest.mdは結構一番最初の、最初に何を渡せばうまくいくかっていうところにも入れておいて、もう訴求しておいてもいいかなとは思ってます。ちょっと流れが全体的に重複する部分とか、ここ最初に言ってないのに途中でもう言ったかのように出てくる、逆もしかりで、ここ言ったのにまた重複して出てくるとか結構あったりするんで、そこら辺しっかりやっておくっていうのが大事かなと思ってます。あとテストのところもちゃんと具体的にどういうテストが必要なのかとかは多分いろいろやってはくれるんだけど、AI側がねコードレビューみたいなコマンドもあったりするんで、そういうのとか使いながらテストしたりとか、コードをリファクタリングして品質を高めていく、あとパフォーマンスを向上していくより軽くするとか、読み込みを早くするみたいなこともテストの品質のところには入れてほしいかなと思ってます。
