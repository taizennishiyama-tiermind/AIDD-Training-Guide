# Azure PoC アプリ基盤 ユーザー向け開発ガイド

## 本ドキュメントの対象者

本ガイドは、VSCode + Claude Codeを使ってPoCアプリを作成する開発者（非IT職）向けのものである。Gitの経験やプログラミングの知識がないことを前提に記載する。

---

## 1. 事前準備

以下のツールが自分のPCにインストールされていることを確認する。未導入の場合は管理者に連絡する。

| ツール | 用途 |
|---|---|
| VSCode | コードエディタ |
| Claude Code | AIによるコード生成（VSCode拡張） |
| Git | バージョン管理 |
| Node.js (v20) | アプリの実行環境 |
| GitHubアカウント | 社用メールで作成済み。Organization に参加済み |

---

## 2. 新しいアプリを作る流れ

### ステップ1：申請する

Google Form（URLは管理者から共有）にアクセスし、以下を入力して送信する。

| 入力項目 | 記入内容 | 例 |
|---|---|---|
| 申請者名 | 自分の名前 | 田中一郎 |
| 社員番号 | 自分の社員番号 | 12345 |
| GitHubユーザー名 | 自分のGitHubアカウント名 | tanaka-ichiro |
| アプリ名 | 英小文字とハイフンで20文字以内 | attendance-checker |
| アプリ論理名 | アプリの日本語名 | 出欠確認ツール |
| アプリ概要 | 何をするアプリか（任意） | 会議の出欠をWebで回収・集計する |

**アプリ名の注意点：**

- 使える文字は英小文字（a-z）、数字（0-9）、ハイフン（-）のみ
- 先頭はハイフン以外にすること
- 20文字以内にすること
- このアプリ名が自動的にURLの一部になる（例: `12345-attendance-checker.azurewebsites.net`）

### ステップ2：完了メールを待つ

申請後、数分〜十数分で完了メールが届く。メールに以下の情報が記載されている。

- **リポジトリURL**: アプリのコードが保存される場所
- **デプロイ先URL**: アプリが公開されるURL
- **cloneコマンド**: 最初にPCに取り込むためのコマンド

### ステップ3：リポジトリをPCに取り込む

VSCodeのターミナル（メニュー → ターミナル → 新しいターミナル）を開き、作業用フォルダに移動してからメールに記載の clone コマンドを実行する。

```
cd ~/Documents
git clone https://github.com/{org}/{物理名}.git
```

完了すると、フォルダが作成される。

### ステップ4：VSCodeでフォルダを開く

VSCode のメニュー → ファイル → フォルダーを開く → 先ほどcloneしたフォルダを選択する。

**重要：必ず1つのアプリにつき1つのVSCodeウィンドウで開くこと。** 複数のアプリフォルダをまとめて開くとClaude Codeが意図しないファイルを変更する可能性がある。

---

## 3. アプリを開発する

### 3-1. Claude Codeで開発を始める

VSCode上でClaude Codeを起動し、自然言語でアプリの内容を指示する。テンプレートには技術スタック（React + Express）の設定が含まれているので、ユーザーが技術的な指定をする必要はない。

**指示の例：**

```
出欠確認ツールを作ってください。
- 会議名、日時、参加候補者のリストを登録できる
- 各参加者がURLにアクセスして出欠を回答できる
- 回答状況を一覧で確認できる
```

### 3-2. ローカルで動作確認する

Claude Codeに「ローカルで起動して」と指示するか、ターミナルで以下を実行する。

```
npm install
npm run dev
```

ブラウザで `http://localhost:3000`（または表示されたURL）にアクセスして動作を確認する。

### 3-3. 修正を繰り返す

動作を確認しながら「ここをこう変えて」とClaude Codeに指示し、再度ローカルで確認する。納得のいく状態になるまで繰り返す。

---

## 4. データの保存について

### 4-1. ローカル開発時

ローカルで開発している間は、JSONファイルやCSVファイルなどのテキストベースのファイルにデータを保存する。Claude Codeにそのように指示すればよい。

**指示の例：**

```
データはJSONファイルに保存するようにしてください
```

この段階では、自分のアプリがどんなデータを扱い、どういう形式が必要かを意識しながら開発を進める。データは必要な形に整理した上でアプリに読み込ませる運用とする。

### 4-2. 本番（デプロイ先）でデータベースを使う

デプロイ先ではデータベース（Cosmos DB）やファイル保存（Blob Storage）が利用できる。環境変数は自動設定されているので、Claude Codeに以下のように指示する。

**指示の例：**

```
本番環境ではCosmos DBにデータを保存するようにしてください。
環境変数 COSMOS_ENDPOINT, COSMOS_KEY, COSMOS_DATABASE が設定されています。
```

ローカルとデプロイ先でデータの保存先が異なる形になるが、この切り替えもClaude Codeに指示して実装してもらえばよい。

---

## 5. アプリを公開（デプロイ）する

### 5-1. プッシュする

Claude Codeに以下のように指示する。

```
変更をGitHubにプッシュしてください
```

Claude Codeが `git add` → `git commit` → `git push` の一連の操作を実行してくれる。

### 5-2. デプロイを待つ

プッシュすると自動的にデプロイが始まる。2〜5分後にデプロイ先URL（メールに記載されたURL）にアクセスすると、アプリが反映されている。

### 5-3. 修正→再デプロイ

以降、修正するたびに「プッシュして」と指示するだけで最新の状態が自動反映される。

```
開発 → ローカル確認 → 「プッシュして」 → 2〜5分後に本番反映
```

この繰り返しが日常の開発サイクルになる。ローカルで十分に動作確認してからプッシュすること。

---

## 6. 問題が起きたとき

### 6-1. デプロイしたアプリが動かない

まずローカルで正常に動作するか確認する。ローカルで動くのにデプロイ先で動かない場合は、管理者に連絡する。

### 6-2. さっきの変更を取り消したい

Claude Codeに以下のように指示する。

```
さっきの変更を取り消してプッシュしてください
```

Claude Codeが `git revert` → `git push` を実行し、前の状態に戻る。

### 6-3. よくわからないエラーが出た

Claude Codeにエラーメッセージを見せて相談する。解決しない場合は管理者に連絡する。

---

## 7. 2つ目以降のアプリを作る場合

ステップ1（Google Form申請）からやり直す。アプリごとにリポジトリとデプロイ先が別々に用意されるので、それぞれ別のフォルダ・別のVSCodeウィンドウで作業する。

---

## 8. 注意事項

- **mainブランチにプッシュすると即座に本番に反映される。** 下書きや試作段階の内容もプッシュすれば公開されるので注意すること。ローカルで十分にテストしてからプッシュする。
- **アプリのURLには社外ネットワーク（出先・スマートフォン等）からもアクセス可能。** ただし、社内のEntra ID認証が自動的に適用されるため、社員アカウントでのMicrosoftログインが必要。社員以外はアクセスできない。
- **不要になったアプリは管理者に連絡して削除を依頼すること。**
